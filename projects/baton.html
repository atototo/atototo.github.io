Baton

AI 에이전트를 조직처럼 운영하기 위한 오케스트레이션 시스템

Baton은 여러 AI 에이전트를 하나의 조직처럼 운영하기 위한 시스템입니다. 단순히 에이전트를 실행하는 도구를 넘어서, 승인 기반 실행 흐름, 워크트리 기반 작업 격리, 비용 통제, 실행 추적, 활동 로그를 포함한 운영 구조를 강화하는 데 집중했습니다. 기존 기반 위에서 실제 운영에 가까운 거버넌스와 실행 워크플로우를 확장한 프로젝트입니다.

01

무엇을 해결하려 했는가

에이전트를 실행하는 것과 운영하는 것은 다른 문제였습니다.

문제

에이전트를 여러 개 붙이면 자동화는 시작되지만, 실제 운영 단계에서는 승인 없이 작업이 진행되거나, 병렬 작업 중 충돌이 나거나, 누가 어떤 작업을 왜 하고 있는지 추적하기 어려운 문제가 생깁니다.

접근

Baton을 단순 실행기가 아니라 운영 시스템으로 확장했습니다. 회사 단위 스코프, 승인 워크플로우, 워크트리 기반 실행 격리, 비용 통제, 활동 로그를 중심으로 구조를 발전시켰습니다.

02

핵심 기여

실제 운영에 필요한 흐름을 제품 기능으로 넣었습니다.

승인 기반 거버넌스 워크플로우 확장

채용, 전략 승인, 실행 흐름에 승인 단계를 반영하고, 리뷰어 에이전트가 개입하는 규칙을 발전시켰습니다.

워크트리 기반 티켓 실행 구조 도입

티켓별 작업 공간을 분리해 병렬 작업 충돌을 줄이고, 작업 단위를 더 안전하게 실행할 수 있는 구조를 만들었습니다.

운영 안정성 보강

Heartbeat 실행 추적, 브랜치 동기화, 충돌 복구 흐름을 추가해 실제 운용 시나리오에 가까운 안정성을 높였습니다.

운영 UI 정리

대시보드, 에이전트, 이슈, 승인, 비용 화면을 연결해 전체 상태를 한눈에 볼 수 있도록 구성했습니다.
