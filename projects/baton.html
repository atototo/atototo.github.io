Baton

AI 에이전트를 조직처럼 운영하기 위한 오케스트레이션 시스템

Baton은 여러 AI 에이전트를 한 팀처럼 일하게 만드는 것에서 끝나지 않고, 누가 어떤 일을 맡고 있는지, 무엇이 허용되는지, 언제 사람이 개입해야 하는지까지 관리하기 위해 만든 프로젝트입니다. 단순한 자동화 도구보다 에이전트 오케스트레이션과 거버넌스 워크플로우에 더 가까운 구조를 지향했고, 그 과정에서 승인 흐름, 작업 격리, 실행 추적, 비용 가시화를 중심으로 제품을 확장했습니다.

GitHub ↗ 가이드 문서 ↗ 프로젝트 목록

왜 만들었는가

에이전트를 여러 개 실행하는 것만으로는 실제 업무 운영이 되지 않았고, 승인과 책임, 진행 상태를 함께 다루는 구조가 필요했습니다.

무엇을 했는가

거버넌스 워크플로우, 워크트리 기반 작업 격리, heartbeat 추적, 비용과 활동 로그를 포함한 운영 흐름을 확장했습니다.

지금도 고민하는 것

에이전트 구성 방식, 컨텍스트 관리, 결과를 평가하고 개선에 반영하는 루프는 계속 발전시키고 있는 주제입니다.

01

왜 만들었는가

에이전트를 실행하는 것과 운영하는 것은 다른 문제였습니다.

운영에서 부딪힌 문제

에이전트를 여러 개 붙이면 자동화는 시작되지만, 실제 운영 단계에서는 누가 어떤 일을 맡고 있는지, 어떤 작업이 승인되어야 하는지, 어디까지 진행됐는지, 충돌이 왜 났는지를 한눈에 보기 어려웠습니다.

Baton의 방향

Baton은 여러 AI 에이전트를 조직처럼 운영하기 위한 시스템으로 설계했습니다. 단순한 작업 보드가 아니라, 오케스트레이션과 거버넌스를 함께 다루는 운영 중심 구조를 목표로 확장했습니다.

02

무엇을 구현했는가

기능을 추가한 것이 아니라 운영 흐름을 제품 기능으로 만들었습니다.

승인 기반 거버넌스 워크플로우 확장

채용, 전략 승인, 작업 진행 흐름에 승인 단계를 반영하고, 리뷰어 에이전트가 개입하는 규칙까지 운영 흐름에 녹였습니다.

워크트리 기반 티켓 실행 구조 도입

티켓별 작업 공간을 분리해 병렬 작업 충돌을 줄이고, 승인된 작업을 더 안전하게 실행할 수 있는 구조를 만들었습니다.

실행 상태와 복구 흐름 보강

Heartbeat 실행 추적, 브랜치 동기화, 충돌 복구 흐름을 추가해 여러 에이전트가 동시에 움직이는 상황을 더 안정적으로 다루도록 했습니다.

운영자가 읽을 수 있는 UI와 문서화

대시보드, 승인, 비용, 활동 로그를 한곳에서 읽을 수 있게 정리하고, 가이드 문서까지 함께 정리해 구조를 설명 가능하게 만들었습니다.

03

무엇이 어려웠는가

일부는 풀었고, 일부는 지금도 계속 고민하고 있습니다.

partially solved

병렬 실행과 작업 격리

여러 에이전트가 동시에 같은 코드베이스를 건드릴 때 충돌 없이 작업을 나누는 것이 핵심 과제였습니다. 워크트리 기반 실행 구조와 충돌 복구 흐름으로 일부 해결했지만, 더 자연스러운 병렬 실행 경험은 계속 다듬고 있습니다.

in progress

에이전트 구성 방식

어떤 역할을 어떤 경계로 나누는 것이 좋은지, 리더와 실행 담당, 리뷰어를 어떻게 조합해야 하는지는 아직도 실험 중입니다. 구조가 바뀌면 운영 방식 전체가 달라지기 때문에 계속 검증하고 있습니다.

in progress

컨텍스트 관리

각 에이전트에게 어떤 맥락을 어디까지 전달해야 하는지는 실행 품질과 비용, 안정성에 모두 영향을 줍니다. 아직 최적의 방식을 확정한 것은 아니지만, Baton에서는 이 문제를 다룰 수 있도록 작업 흐름, 실행 기록, 문서 구조를 먼저 정리하며 기반을 만들고 있습니다.

in progress

평가와 개선 루프

에이전트가 잘 일했는지 어떻게 평가하고, 그 결과를 다음 실행과 구성 방식에 어떻게 반영할지는 아직 본격적으로 풀고 있는 단계는 아닙니다. 대신 Baton Dashboard, 실행 로그, 상태 시각화처럼 이후 평가와 개선으로 이어질 수 있는 운영 기반을 먼저 만들고 있습니다.

04

어떤 문제를 해결하고 있는가

Baton은 에이전트를 단순 실행하는 도구보다, 운영 구조를 만드는 데 더 가깝습니다.

책임과 상태가 보이는 구조

누가 어떤 작업을 맡고 있는지, 어디까지 진행됐는지, 어떤 상태인지 추적 가능한 흐름을 만들고 있습니다.

사람이 개입할 지점 명확화

승인이 필요한 순간과 자동 실행 가능한 순간을 구분해, 운영자가 통제할 수 있는 구조를 강화하고 있습니다.

지속 가능한 운영 기반

비용, 활동 로그, 실행 기록을 함께 보게 만들어 에이전트 자동화를 일회성 데모가 아니라 운영 가능한 형태로 바꾸고 있습니다.
